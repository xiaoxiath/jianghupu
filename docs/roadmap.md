# 🗺️ 《江湖残卷》项目路线图与里程碑

---

## 1. 总体目标

将《江湖残卷》从概念阶段推进到可玩的 V1.0 版本，并为未来的社区驱动扩展（MOD）奠定基础。

---

## 2. 版本规划 (高阶)

| 版本 | 核心目标 | 状态 |
|---|---|---|
| **V0.1** | **可玩核心循环** (Playable Core Loop) | `已完成` |
| **V0.3** | **动态叙事整合** (Dynamic Narrative) | `已完成` |
| **V0.5** | **活态世界引擎** (Living World Engine) | `已完成` |
| **V0.8** | **无限轮回机制** (Infinite Replayability) | `已完成` |
| **V1.0** | **公开发行版** (Public Release) | `进行中` |

---

## 3. V0.1: 可玩核心循环

- **目标**: 实现一个包含基础探索、战斗和角色成长的 CLI Demo。
- **验收标准**:
    - [x] 玩家可以创建角色。
    - [x] 玩家可以在一个静态生成的地图上使用 `go` 命令移动。
    - [x] 地图上有可交互的 NPC，可以触发固定战斗。
    - [x] 战斗系统采用回合制，有胜负判断。
    - [x] 战斗胜利后可以获得经验并升级。
- **关键任务 (Epics)**:
    - [x] **E01**: 搭建项目基础框架 (TypeScript, pnpm, esbuild)。
    - [x] **E02**: 实现 CLI 交互层 (inquirer, chalk)。
    - [x] **E03**: 开发基础角色属性与状态管理系统。
    - [x] **E04**: 开发回合制战斗引擎。
    - [x] **E05**: 实现基于 `seed` 的静态地图生成。

---

## 4. V0.3: 动态叙事整合

- **目标**: 引入 AI 说书人，让游戏从“机械”变得“有机”。
- **验收标准**:
    - [x] 游戏的主要文本由 AI 说书人生成。
    - [x] 玩家的行动由 AI 提供的选项驱动。
    - [x] 引入随机事件和基础奇遇。
- **关键任务 (Epics)**:
    - [x] **E06**: 集成 LLM 接口 (本地 Ollama 或云端 API)。
    - [x] **E07**: 实现 AI 说书人状态机和 Prompt 管理。
    - [x] **E08**: 开发随机事件触发系统。

---

## 5. V0.5: 活态世界引擎

- **目标**: 让 NPC 和门派“活”起来，世界会自行演化。
- **验收标准**:
    - [x] NPC 会在世界中自主移动、修炼和交互。
    - [x] 门派之间会发生关系变化（结盟、敌对）。
    - [x] 引入动态的全局事件（如“魔教入侵”）。
- **关键任务 (Epics)**:
    - [x] **E09**: 开发 NPC 行为引擎。
    - [x] **E10**: 开发门派势力演化系统。
    - [x] **E11**: 实现全局动态事件系统。

---

## 6. V0.8: 无限轮回机制

- **目标**: 实现游戏的核心 Roguelike 特性——传承与无限游玩。
- **验收标准**:
    - [x] 玩家角色死亡后，可以创建新角色并继承部分遗产（功法、声望等）。
    - [x] AI 说书人能够讲述前代角色的故事。
    - [x] 世界状态可以被快照和存档。
- **关键任务 (Epics)**:
    - [x] **E12**: 开发传承系统 (Legacy System)。
    - [x] **E13**: 实现游戏存档/读档与世界快照功能。

---

## 7. V1.0: 公开发行版

- **目标**: 优化、打包并发布游戏。
- **验收标准**:
    - [ ] 游戏内容和数值达到平衡。
    - [ ] 性能优化，无明显卡顿。
    - [ ] 打包为跨平台可执行文件。
    - [x] 完善的 `README.md` 和上手指南。
- **关键任务 (Epics)**:
    - [x] **E14**: 游戏内容扩展与文本润色。
    - [ ] **E15**: 性能分析与优化。
    - [x] **E16**: 实现 MOD 加载机制。
    - [ ] **E17**: 打包与发布流程。
